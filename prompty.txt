Prompt 1 — bootstrap
Create a new open-source Python desktop application for astrology.

Requirements:
- Python 3.12
- PySide6 GUI
- SQLite storage
- pytest for tests
- Ruff for linting/formatting
- clean modular architecture
- AGPL-3.0 licensed project

Project goal:
Build a desktop astrology application that will eventually replace Astro-Seek for a specific workflow.
The first priority is natal chart support and later transit aspect date-range search.

Create this structure:

astroapp/
  app/
    main.py
    config/settings.py
    models/
    engine/
      ephemeris/
      natal/
      transit/
    services/
    storage/
    ui/
      widgets/
    utils/
  tests/
  docs/
  pyproject.toml
  README.md
  LICENSE

Tasks:
1. Generate the full project skeleton.
2. Add pyproject.toml with dependencies and dev dependencies.
3. Configure Ruff.
4. Configure pytest.
5. Add a minimal PySide6 app that opens a main window.
6. Add a minimal SQLite setup layer.
7. Add README with architecture summary and roadmap.
8. Add AGPL-3.0 license file.
9. Add GitHub Actions workflow for lint + tests.

Important:
- Keep business logic out of the UI.
- Use type hints everywhere.
- Keep modules small and focused.
- Do not implement astrology calculations yet.

Prompt 3 — Swiss backend
Extend the astrology desktop application with an ephemeris backend abstraction.

Goal:
Prepare the system for Swiss Ephemeris integration through pysweph.

Implement:
1. EphemerisBackend protocol/interface
2. SwissEphemerisBackend class
3. functions for:
   - get planet longitude for UTC datetime
   - get house cusps for UTC datetime and coordinates
   - get ascendant and MC
4. configuration for ephemeris data path
5. clear TODOs and adapter boundaries so the rest of the app does not depend directly on pysweph internals

Important:
- isolate pysweph usage in one module
- keep output normalized and typed
- add tests for backend shape and utility functions
- do not implement full UI here

Prompt 6 — zakresy dat aspektów
Implement transit-to-natal aspect date-range search.

Goal:
For a selected person and date range, detect transit aspect events with:
- start datetime when entering orb
- exact datetime
- end datetime when leaving orb

Scope:
- major aspects only
- configurable orb
- selected transit bodies
- selected natal bodies

Implement:
- transit_position_sampler.py
- aspect_scanner.py
- event_refiner.py
- transit_service.py

Output:
AspectEvent records with:
- transit body
- natal body
- aspect type
- start_dt
- exact_dt
- end_dt
- exact_orb
- applying/separating

Add tests for:
- never enters orb
- enters and exits orb
- exact occurs inside the range
- starts already in orb
- ends after the search window
